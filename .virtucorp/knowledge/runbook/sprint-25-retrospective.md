### 3. 自动化交易调度器 (Issue #329)

**核心价值**: 实现 24/7 自动化交易，满足专业交易者需求。

**功能特性**:

- 完整的执行历史记录
- 安全配置: 仓位限制、止损止盈、熔断机制
- 手动触发执行

**技术实现**:
- 数据库: `trading_schedules`、`schedule_executions`、`schedule_safety_config`
- 服务层: `SchedulerService` 基于 `cron` npm 包
- 前端: `SchedulerPage.tsx` 使用 Arco Design

**安全机制**:
- 最大仓位大小和百分比限制
- 每日交易次数和金额限制
- 止损/止盈自动触发
- 连续失败自动暂停

## 三、遇到的问题和解决方案

### 问题 1: AI 助手 LLM 多后端集成

**挑战**: 需要支持 OpenAI、Anthropic、Ollama 三种不同 API 格式。

**解决方案**: 
- 设计统一的 `LLMProvider` 接口
- 实现工厂模式，根据环境变量动态选择后端
- 配置化模型选择，便于切换

### 问题 2: 调度器时区和并发处理

**挑战**: Cron 任务需要考虑用户时区，且需要防止并发执行冲突。

**解决方案**:
- 存储用户时区偏好
- 执行前检查任务状态，防止重复执行
- 添加执行锁和超时机制

### 问题 3: 订阅系统与现有功能集成

**挑战**: 需要在不破坏现有功能的前提下添加权限控制。

**解决方案**:
- 中间件设计，最小化代码改动
- 功能降级而非硬性拦截（如限制次数而非完全禁止）
- 提供升级提示 UI 组件
