## 🐛 遇到的问题和解决方案

### 问题 1: 数据库迁移设计
**描述**: 策略模板市场需要设计多张关联表，初期对表结构设计有争议。

**解决方案**: 
- 采用三表设计: `strategy_templates`（主表）、`template_ratings`（评分）、`template_usage`（使用记录）
- 使用 JSON Schema 验证策略参数，保持灵活性
- 添加触发器自动计算平均评分

### 问题 2: 多时间框架数据缓存策略
**描述**: 不同时间框架的数据更新频率不同，缓存策略需要差异化。

**解决方案**:
- 实现 timeframe-aware TTL: 缓存时间与时间框架长度成正比
- 1m 数据缓存 60s，1h 数据缓存 360s，1d 数据缓存 8640s
- 提供缓存统计和清理 API

### 问题 3: TWAP 订单切片逻辑
**描述**: TWAP 订单需要在指定时间范围内均匀切片执行。

**解决方案**:
- 后端计算切片数量和间隔
- 每个切片记录独立状态
- 前端实时显示执行进度
