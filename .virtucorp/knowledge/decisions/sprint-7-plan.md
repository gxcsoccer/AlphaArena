1. **测试覆盖** - 为 K线图表添加单元测试和 E2E 测试
2. **日志优化** - 建立统一的日志系统
3. **质量保障** - 防止 Bug 回归

## 详细规划

### Issue #172: K线图表单元测试

**测试范围：**
- 图表初始化测试（挂载、loading 状态、卸载清理）
- 数据更新测试（初始数据、更新、等待就绪）
- Symbol 切换测试（清除旧数据、验证匹配、忽略过期响应）
- 竞态条件测试（快速切换、卸载后响应）

**技术要求：**
- Vitest + React Testing Library
- Mock lightweight-charts 库
- Mock useKLineData hook
- 测试覆盖率 > 80%

### Issue #173: 结构化日志系统

**目标：**
- 统一日志格式：`[Module] LEVEL: message {data}`
- 日志级别：DEBUG, INFO, WARN, ERROR
- 环境区分：开发环境 DEBUG，生产环境 WARN
- 模块化：每个模块独立 Logger 实例

**迁移计划：**
1. 创建 Logger 工具类
2. 迁移 K线图表模块日志
3. 迁移订单簿模块日志
4. 添加单元测试

### Issue #174: E2E 测试（交易对切换）

**测试场景：**
- 基本流程：切换交易对 → 验证数据正确
- 快速切换：连续点击 → 验证最终数据匹配
- 网络延迟：慢速网络 → 验证加载状态

**技术方案：**
- 使用现有验收测试框架
- 集成到 CI 流程
