### 2. Supabase 私有 API 依赖风险 (#187 → #190)

**问题**: Sprint 8 使用了 Supabase 私有 API `_off` 方法，存在技术债务风险

**解决方案**:
- 创建类型安全的抽象层 `src/shared/realtime/listener-utils.ts`
- 提供 `removeChannelListener()`、`createBroadcastUnsubscribe()` 等工具函数
- 集中管理私有 API 使用，便于未来迁移
- 清晰的 JSDoc 文档说明风险和理由

**收益**:
- 类型安全：不再需要 `(channel as any)` 类型断言
- 集中管理：Supabase API 变更时只需更新一处
- 可测试性：更容易 mock 和测试

**代码变更**: +206 行 / -17 行  
**合并时间**: 2026-03-16
