# AlphaArena 产品分析：用户动线梳理与功能整合建议

> 分析日期：2026-03-23
> 分析背景：用户反馈当前功能很散、不清晰，需要完整考虑用户操作动线和解决用户实际需求

## 一、现状诊断

### 1.1 页面数量统计

当前产品共有 **35+ 个独立页面/路由**，侧边栏菜单显示 **27 个一级菜单项**：

| 分类 | 页面 |
|------|------|
| **交易相关** | 行情、Trades、Holdings、Advanced Orders、Virtual Account、Exchange Accounts |
| **策略相关** | Strategies、Strategy Comparison、Strategy Portfolio、Strategy Marketplace、Copy Trading |
| **分析相关** | Dashboard、User Dashboard、Performance、Attribution、Risk、Sentiment、Backtest |
| **账户管理** | Subscription、Notification Preferences、Data Source、Scheduler |
| **其他** | API Docs、Leaderboard、Journal、Rebalance |

### 1.2 核心问题诊断

#### 问题 1：菜单层级过深
- 所有功能平铺在一级菜单，没有层级分组
- 用户需要扫描 27 个选项才能找到目标功能
- 缺乏信息架构的层次感

#### 问题 2：功能重复/分散
- **Dashboard vs User Dashboard**: 两个仪表板功能重叠
- **Performance vs Attribution**: 绩效相关功能分散
- **Trades vs Journal**: 交易历史和交易日志功能重叠
- **Strategies vs Strategy Portfolio vs Strategy Comparison vs Marketplace**: 策略功能分散在 4 个页面

#### 问题 3：术语混乱
- 混合使用中英文：`Strategies`、`Trades`、`Holdings` vs `交易日志`、`绩效`
- 缺乏统一的命名规范

#### 问题 4：核心用户旅程不清晰
- 用户不知道从哪里开始
- 没有明确的"开始交易"或"创建策略"入口
- 各功能之间缺乏引导和关联

## 二、用户动线分析

### 2.1 目标用户画像

| 用户类型 | 描述 | 核心需求 |
|----------|------|----------|
| **量化爱好者** | 对量化交易感兴趣，希望学习和实践 | 简单易用的模拟环境、学习资源 |
| **独立交易者** | 有一定经验，希望测试和优化策略 | 策略开发工具、回测能力 |
| **量化开发者** | 专业开发者，需要 API 和高级功能 | API 文档、高性能执行 |

### 2.2 用户实际需求

**核心需求**：在零风险环境下验证交易策略

**用户真正想解决的问题**：
1. "我想验证我的交易想法是否可行" → 策略创建 + 回测
2. "我想看看我的策略表现如何" → 绩效分析
3. "我想改进我的策略" → AI 助手 + 优化建议
4. "我想学习量化交易" → 教程 + 模拟实战

### 2.3 当前用户旅程（问题版）

```
用户进入 → 看到行情页面 → 不知所措 → 点击菜单 → 看到 27 个选项 → 困惑
     ↓
尝试点击 Strategies → 发现可以创建策略 → 但不知道如何回测
     ↓
点击 Backtest → 不知道如何关联到策略 → 放弃
```

**断点识别**：
1. **入口断点**：Landing Page → 交易界面，没有过渡
2. **导航断点**：功能之间缺乏关联引导
3. **流程断点**：策略创建 → 回测 → 分析 没有连贯流程

### 2.4 理想用户旅程

```
新用户旅程：
Landing Page → 注册 → 新手引导 → 模拟账户初始化 → 策略模板库 → 选择策略 → 运行回测 → 查看结果 → 优化建议

老用户旅程：
登录 → 我的仪表板（概览） → 快速操作（运行策略/下单/查看绩效） → 进入具体功能模块
```

## 四、实施建议

### 4.1 第一阶段：菜单重构（P0）

**目标**：将 27 个一级菜单项减少到 5-6 个

**具体任务**：
1. 实现菜单分组（使用 ArcoDesign Menu SubMenu）
2. 保留现有路由，只改变菜单结构
3. 添加快捷入口（常用功能）

**预期效果**：
- 用户扫描时间减少 70%
- 导航清晰度提升

### 4.2 第二阶段：Dashboard 整合（P0-P1）

**目标**：打造清晰的用户首页

**具体任务**：
1. 合并 Dashboard 和 User Dashboard
2. 添加 Tab 切换（总览/绩效/风险/情绪）
3. 添加"快速操作"入口
4. 添加"新手引导"触发器

### 4.3 第三阶段：功能模块化（P1-P2）

**目标**：将相关功能整合为三大模块

**具体任务**：
1. 策略中心：创建 Tab 式策略管理界面
2. 交易中心：整合交易相关功能
3. 设置中心：整合账户和配置功能

### 4.4 第四阶段：用户旅程优化（P2-P3）

**目标**：优化新用户入门体验

**具体任务**：
1. 新手引导流程（5 步完成首次策略运行）
2. 空状态引导
3. 功能关联推荐
